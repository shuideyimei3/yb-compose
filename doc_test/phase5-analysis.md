## 分析

三次结果显示，jitter+loss 对尾延迟的影响不完全稳定：Run 1 出现明显秒级 P99，Run 2 和 Run 3 则较平稳。这符合丢包类故障的概率特征。样本量较小时，是否撞上 TCP 重传和 Raft 路径重试，会显著影响 P99。

带宽限制到 10mbit 对本实验的小查询影响不大。读写 payload 很小，瓶颈仍是 RTT、重传和分布式协调，而不是吞吐带宽。

## 结论

跨 region 场景中，平均延迟主要由基础 RTT 决定；P99 则对丢包和 jitter 更敏感。生产环境优先应控制丢包和链路稳定性，而不是只关注带宽数值。
