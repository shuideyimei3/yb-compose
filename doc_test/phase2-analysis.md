## 实验目的

在无额外 `tc netem` 延迟的 5 节点 RF=3 集群上，测量各节点 YSQL 简单读写延迟，作为跨区域延迟实验的基线。

## 分析

无额外网络延迟时，各节点读写延迟集中在 88-91ms，节点间差异很小。这个基线并非单机 PostgreSQL 延迟，而是 YugabyteDB YSQL + DocDB + Raft + Docker 网络栈的组合延迟。

读写延迟接近，说明在该脚本的简单事务模型下，客户端连接、YSQL 层处理和分布式 tablet 路由占了主要固定成本。写入仍需要 Raft 多数派确认，但同主机 Docker 网络下跨节点 RTT 很低，因此没有形成数量级差异。

## 结论

5 节点 RF=3 基准环境稳定，简单读写平均延迟约 90ms。后续延迟注入实验中，读写延迟超过该基线的部分主要来自人为注入的跨 region egress 延迟和 Raft 通信路径。
